=== sim ===

This directory contains testbench files for simulation. Testbenches are used to verify the functionality and timing of individual modules or the whole system in Vivado Simulator before synthesis.

本目录存放仿真测试文件（Testbench）。这些文件用于在Vivado仿真器中综合前验证各模块或整个系统的功能与时序是否正确。

=== Note on clk_wiz (Clock Module) ===

The clock module (clk_wiz.v) is generated by Vivado's IP Catalog (Clocking Wizard) and therefore has NO testbench in this directory. The IP core is vendor-provided and does not require RTL simulation — its behavior is verified by Vivado's IP-level simulation models automatically included when the project is set up in Vivado.

When simulating other modules that depend on a clock (e.g., tb_game_logic.v, tb_vga_render.v), the testbench should either:
1. Generate a 25.175 MHz clock directly in the testbench (preferred for simulation speed and independence from Vivado IP), or
2. Instantiate the clk_wiz wrapper after ensuring the Vivado IP simulation library is loaded.

Approach 1 is used in the existing testbenches in this directory.

=== 关于 clk_wiz（时钟模块）的说明 ===

时钟模块（clk_wiz.v）由 Vivado IP Catalog 的 Clocking Wizard 生成，因此本目录中**没有**与之对应的仿真测试文件。该 IP 核由厂商提供，无需进行 RTL 仿真 —— 在 Vivado 中正确搭建项目后，其仿真模型会被自动包含。

在仿真其他依赖时钟的模块时（如 tb_game_logic.v、tb_vga_render.v），测试文件应选择以下方式之一提供时钟：
1. 直接在 testbench 中生成 25.175 MHz 时钟（推荐，仿真速度快且不依赖 Vivado IP），或
2. 在确认 Vivado IP 仿真库已加载的前提下，实例化 clk_wiz 包装器。

本目录中的现有 testbench 均采用方式 1。
